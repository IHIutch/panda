# Using Panda in a Design System

Panda is a great fit for design systems. Using the presets, you can build a library of reusable layout patterns, tokens, and recipes that can be used across multiple projects.

Let's say you have a monorepo with `app` and `design-system` that matches the following structure:

<FileTree>
  <FileTree.Folder name="app" defaultOpen>
    <FileTree.File name="package.json" />
  </FileTree.Folder>
  <FileTree.Folder name="design-system" defaultOpen>
    <FileTree.File name="package.json" />
    <FileTree.File name="tsup.config.ts" />
  </FileTree.Folder>
</FileTree>

## Configuring Panda

Follow the [installation instructions](/docs) to install Panda in both projects.

Your project should now look like this:

<FileTree>
  <FileTree.Folder name="app" defaultOpen>
    <FileTree.File name="panda.config.ts" />
    <FileTree.File name="postcss.config.cjs" />
    <FileTree.File name="package.json" />
  </FileTree.Folder>
  <FileTree.Folder name="design-system" defaultOpen>
    <FileTree.File name="panda.config.ts" />
    <FileTree.File name="postcss.config.cjs" />
    <FileTree.File name="package.json" />
    <FileTree.File name="tsup.config.ts" />
  </FileTree.Folder>
</FileTree>

## Setup Design System

In the design system directory, update the panda config as follows:

- Set the `emitPackage` option to `true` in your config
- Set the `outdir` to any package name you want to use

```ts filename="design-system/panda.config.ts"
export default defineConfig({
  // ...
  emitPackage: true,
  outdir: 'style-engine'
})
```

This option will write the panda output to the specified package name in the `node_modules/style-engine`

### Create Preset

You can now define presets in the design system project. For example, let's create a preset that includes some basic colors.

```ts filename="design-system/src/panda.preset.ts"
import { definePreset } from '@pandacss/dev'

export const preset = definePreset({
  theme: {
    tokens: {
      colors: {
        primary: { value: 'blue' },
        secondary: { value: 'green' }
      }
    }
  }
})
```

Now, you can add the preset to the `panda.config.ts` file:

```ts filename="design-system/panda.config.ts"
import { defineConfig } from '@pandacss/dev'

export default defineConfig({
  // ...
  presets: ['panda.preset.ts']
})
```

### Generate artifacts

Delete the initially generated `styled-system` directory in both project, and run the following command in both projects:

```bash
pnpm panda codegen
```

### Export Preset

Then you can export the preset in the `index.ts` file:

```ts filename="design-system/src/index.ts"
export { preset } from './panda.preset'
```

### Start using Panda

You can now import the generated artifacts in your project:

```tsx filename="design-system/src/index.tsx"
import { css } from 'style-engine/css'

export default function App() {
  return <div className={css({ fontSize: 'lg' })}>Hello World</div>
}
```

### Mark as External

If you want to build the design system, make sure you mark it as an external in your bundler.

For example, in tsup, you can do this like so:

```js filename="design-system/tsup.config.ts"
import { defineConfig } from 'tsup'

export default defineConfig({
  external: ['style-engine']
})
```

## Setup App

In the app directory, update the panda config as follows:

- Set the `emitPackage` option to `true` in your config
- Set the `outdir` to the same package name you used in the design system
- Add the preset from the design system package

```ts filename="app/panda.config.ts"
import { defineConfig } from '@pandacss/dev'
import { preset } from '@acme-org/design-system'

export default defineConfig({
  // ...
  presets: [preset]
  emitPackage: true,
  outdir: 'style-engine'
})
```

### Generate artifacts

Delete the initially generated `styled-system` directory in both project, and run the following command in both projects:

```bash
pnpm panda codegen
```

### Start using Panda

You can now import the generated artifacts in your project:

```tsx filename="app/src/main.tsx"
import { css } from 'style-engine/css'

export default function App() {
  return <div className={css({ fontSize: 'lg' })}>Hello World</div>
}
```

## Shipping components

Until now, we've learnt to easily share theme tokens across projects using presets. But what about components?
For example, let's say you have a `Button` component in the design system:

```tsx filename="design-system/src/Button.tsx"
import { css } from 'style-engine/css'

export function Button({ children }) {
  return <button className={css({ color: "blue.300", fontSize: "xl", fontWeight: "bold" })}>{children}</button>
}
```

If you try to import this component in the app, you will notice that even though the utilities classes are correclty attached to the button in the DOM, there is no corresponding CSS generated.
This is because when you run the `panda` process in your app, it is not aware that the `Button` component is using any tokens.

```tsx filename="app/src/main.tsx"
import { Button } from '@acme-org/design-system'

export default function App() {
  return <Button>Click me !</Button>
}
```

To fix this, you could include the design system source files within the `panda.config.ts` of your app:

```ts filename="app/panda.config.ts"
import { defineConfig } from '@pandacss/dev'

export default defineConfig({
  // ...
  include: [
    './src/**/*.{ts,tsx,jsx}',
    './node_modules/@acme-org/design-system/src/**/*.{ts,tsx,jsx}'
  ],
})
```

But this is not ideal performance-wise, as it will search for all tokens usage in those static files on each build.
You might not even have access to the source files if you are using someone else's library published on npm.

Instead, as a design system author, you should run the following command:

<Tabs items={['pnpm', 'npm', 'yarn']}>
  <Tab>
    ```bash
    pnpm panda ship --outfile dist/panda.json
    ```
  </Tab>
  <Tab>
    ```bash
    npm panda ship --outfile dist/panda.json
    ```
  </Tab>
  <Tab>
    ```bash
    yarn panda ship --outfile dist/panda.json
    ```
  </Tab>
</Tabs>

This will generate a file containing all of the tokens usage extracted from your design system components.
This file should be published alongside the rest of your design system files so that consumers can then include this file in their `panda.config.ts`:

```ts filename="app/panda.config.ts"
import { defineConfig } from '@pandacss/dev'

export default defineConfig({
 // ...
  include: [
    './src/**/*.{ts,tsx,jsx}',
    './node_modules/@acme-org/design-system/dist/panda.json'
  ],
})
```

You should now see the correct styles applied to the button in your app ! ðŸŽ‰
